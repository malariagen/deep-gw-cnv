Hi Chiyun,

Experiment 05 results summary:
- PM2_PM3: FNR=0.98 — near-total failure; signal clearly present (FN p50 CRR=1.47)
  but Viterbi discards it
- GCH1: FNR=0.29, FN p50 CRR=1.18 ≈ 1.0 — weak latent signal, not just HMM
- CRT: FNR=0.04, PPV=1.00, MCC=0.98 — near-perfect; bin-count gate working well
- MDR1: FNR=0.11, PPV=1.00 — FPs eliminated; some FNs remain

My diagnosis:
The core problem is training-distribution bias: PM2 amplification is present in a large
fraction of Pf9 samples (~50%), so the VAE learns amplified coverage as the default state.
GCH1 FN p50 CRR=1.18 ≈ 1.0 confirms the latent representation barely distinguishes
amplified from normal. Downsampling CNV-positive samples directly fixes this without
touching any other knob.

Proposed experiment 06 (revised x3): CNV-sample downsampling only
Changes from 05:
- cnv_downsample_ratio: 0.25 (new)
  CNV-positive samples (any gene, any call type) are drawn at 1/4 the rate of
  normal samples during training via WeightedRandomSampler. Rebalances the effective
  training distribution so the model encodes amplification as the exception, not the
  baseline.
- max_beta: 1.0 (unchanged from exp 05)
- warmup_epochs: 50 (unchanged from exp 05)
- hmm_self_transition: 0.75 (unchanged)

This is a clean ablation — one variable changed, everything else held constant.

What I expect:
PM2_PM3 should see the largest benefit — it is the most common CNV in Pf9 and therefore
the biggest source of distribution bias. GCH1 should also improve as the latent space
stops conflating amplified with normal. CRT and MDR1 should remain stable. Main risk:
CNV-positive samples appear less often per epoch, which could slightly degrade their
reconstruction quality — monitor PPV alongside FNR.

What we could do instead:
1. Add max_beta 1.0 → 4.0 on top of downsampling — if downsampling alone is insufficient,
   stronger KL regularisation could further clean up the latent space.
2. Lower hmm_self_transition (0.75 → 0.65) — directly addresses PM2_PM3 Viterbi bottleneck;
   no retraining, fast to test if the VAE fix underdelivers.

Run time estimate: Full VAE retrain (~3–4 hrs on Mac mini MPS) + HMM ~25 min + eval ~2 min.
